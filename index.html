Команди в git

Головна | Галерея
Команда ----- Опис
git init project_name Де project_name назва проекту Створює новий локальний репозиторій з заданим ім’ям
git clone url - адрес Отримання репозиторія за його url - адресою Скачає репозиторій та всю його історію змін
git branch branch_name Ім’я нової гілки branch_name Створення нової гілки
git checkout –b branch_name Ім’я нової гілки branch_name Спосіб створити нову гілку та одразу перейти в неї
git branch ------ Щоб подивитись список усіх гілок
git branch –d branch_name Ім’я гілки яку видаляють branch_name Щоб видалити відповідну гілку
git branch -m old_branch_name new_branch_name Стара назва old_branch_name нова назва new_branch_name Перейменування гілки
git add . ------ Коли потрібно додати до відстеження усі файли
git add file_path Де file_path це шлях до файлу Якщо один файл. Індексує вказаний файл для подальшого комміта
git commit ------ Для того, щоб “зберегти” файли в Git
git commit -m “commit_message” Де “commit_message” – це ваше повідомлення, воно обов’язково повинно бути в лапках Фіксує проіндексовані зміни і зберігає їх в історію версій та додає повідомлення для комміту
git stash Зберігання та відновлення не завершених змін Тимчасова зберігає всі незафіксовані зміни у відслідковуваних файлів
git stash pop Зберігання та відновлення не завершених змін Відновлює стан раніше збережених версій файлів
git stash list Зберігання та відновлення не завершених змін Якщо потрібно вивести список всіх тимчасових зберігань
git stash pop stash@{2} Де stash@{2} це вказівник на елемент списку відкладених змін Якщо потрібно дістати та застосувати якийсь конкретний елемент списку
